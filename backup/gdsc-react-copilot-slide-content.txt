SLIDE 06 ? What is React?
========================================================================
What is React?

The 60-second version for today.

React is a JavaScript library

It helps you build user interfaces from reusable pieces called components.

What this means

React updates the screen automatically when your data changes. You write less boilerplate code.

Used by product teams

Instagram
Netflix
Airbnb

You are learning a workflow used in real web products, not a toy example.

SLIDE 07 ? Why React?
========================================================================
Why React?

React flips the UI update model from manual changes to declarative state.

Before React

Manually update the page

button.onclick = () => {
count++;
label.innerText = count;
}

You find elements and tell the browser exactly what to change.

With React

Update state, React updates UI

setCount(count + 1)

<p>{count}</p>

You describe the UI for the current data. React handles the screen update.

Remember

Declarative UI means you write what you want to see based on state. You update the state, and React handles the DOM.

React Component Mental Model

A component is a reusable UI block. For this app, the tree is simple.

Remember

A component is just a JavaScript function that returns UI. Build it once, pass different props, and reuse it anywhere.

SLIDE 09 ? JSX
========================================================================
JSX

JSX looks like HTML, but it is written inside JavaScript.

What this means

JSX looks like HTML, but it lives inside JavaScript. React compiles it into standard HTML elements.

Rules of JSX

{ } Use curly braces for JavaScript variables or logic

class Use className instead of class for CSS styling

</> All tags must be closed explicitly (e.g. <input />)

1 Must return a single parent element (or a fragment <>)

App.jsx

function App() {
const name = "Alya";

return (
<main>
<h1 className="header-title">Student Task Manager</h1>
<p>Built by {name}</p>
</main>
);
}

SLIDE 11 ? Components
========================================================================
Components

A component is a reusable JavaScript function that returns UI.

Remember

A component is just a JavaScript function that returns UI. It must begin with a capital letter (e.g. Header) and return JSX.

Capitalization rule

Header is a React component. header is a built-in HTML tag.

Header.jsx

<h1>Task Manager</h1>

export default Header;

SLIDE 15 ? State: React's Memory
========================================================================
State: React's Memory

State stores data that can change and trigger a re-render.

What this means

State is React’s memory. When state variables change, React automatically re-renders the component to reflect new data on the screen.

tasks current value

setTasks update function

[] starting value

Remember

Never mutate state arrays or objects directly (e.g. tasks.push() ). Always use the setter function (e.g. setTasks() ) to guarantee re-rendering.

App.jsx

import { useState } from 'react';

const [tasks, setTasks] = useState([]);

// tasks = current value
// setTasks = update function
// [] = starting value

SLIDE 18 ? Add Task Feature
========================================================================
Add Task Feature

Controlled input plus a state update.

What this means

The input value lives in state. When the user types, state updates. When they add, tasks updates.

Remember

The spread operator creates a new array instead of modifying the old one.

App.jsx

const [newTask, setNewTask] = useState("");

const handleAddTask = () => {
if (!newTask.trim()) return;
setTasks([
...tasks,
{
id: Date.now(),
title: newTask.trim(),
status: 'Pending',
priority: 'medium'
}
]);
setNewTask("");
};

SLIDE 19 ? Delete Task Feature
========================================================================
Delete Task Feature

Remove an item without mutating the original array.

Remember

filter() creates a new array without the deleted task.

Data flow

Pass handleDeleteTask down as a prop so TaskCard can trigger the parent update.

delete-task.jsx

const handleDeleteTask = (id) => {
setTasks(
tasks.filter(task => task.id !== id)
);
};

<TaskCard
onDelete={handleDeleteTask}
/>

SLIDE 20 ? Complete Task + Counter
========================================================================
Complete Task + Counter

Two connected ideas: update one task, then calculate live counts from the updated array.

Toggle Complete

handleToggleTask

const handleToggleTask = (id) => {
setTasks(tasks.map(task =>
task.id === id
? { ...task, status: task.status === 'Done'
? 'Pending' : 'Done' }
: task
));
};

Remember
map() checks every task, but only changes the selected one.

Derived Task Counter

5
Total

2
Pending

3
Done

counts + Header props

const pendingCount = tasks.filter(t =>
t.status === 'Pending').length;
const doneCount = tasks.filter(t =>
t.status === 'Done').length;
const totalCount = tasks.length;

<Header pendingCount={pendingCount}
doneCount={doneCount} totalCount={totalCount} />

Remember
The counter is derived from existing state, so it stays in sync automatically.

SLIDE 24 ? Writing Better Prompts
========================================================================
Writing Better Prompts

Give Copilot specific, high-quality context for better suggestions.

Weak Prompt

// add filter

Too brief. Copilot does not know the inputs, behavior, or constraints.

Strong Prompt

// Add a search bar that filters tasks by title and status in real time. Keep the original tasks array unchanged.

Clear task, inputs, expected behavior, and constraint.

What to build (e.g. Search input)
Inputs (e.g. query state)
Expected behavior (e.g. filter array)
Constraints (e.g. case insensitive)

Key Takeaways

Your take-home cheat sheet for the next React project.

React Foundations

JSX UI syntax written inside JavaScript.

Component A function that returns reusable UI.

Props Read-only data from parent to child.

State Data that triggers a re-render when updated.

useState The hook that creates state and its setter.

Arrays Use map, filter, and spread for new arrays.
